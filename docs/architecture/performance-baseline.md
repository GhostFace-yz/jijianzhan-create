# 极简栈创造 v1.0 性能基线

> 状态：已冻结（v1.0）  
> 适用范围：前端、后端、AI 生成、媒体处理、队列  > 决策日期：2026-06-18  
> 决策人：Blueprint · 架构师

---

## 1. 总则

本性能基线是 v1.0 所有模块开发与验收的最低可接受标准。指标分为**用户体验指标**、**API 性能指标**、**AI 生成指标**、**资源与稳定性指标**四类。

所有指标在以下基准环境下测量：

- **桌面端**：Chrome 最新版，1920×1080，8 核 CPU，16GB 内存。
- **低配端**：Chrome 最新版，1366×768，4 核 CPU，8GB 内存。
- **后端**：Node.js 22 LTS，4 核 CPU，8GB 内存，SSD。
- **网络**：100 Mbps，延迟 ≤50ms。

---

## 2. 前端性能指标

| 指标 | 目标值 | 测量方式 | 责任人 |
|------|--------|----------|--------|
| 首屏加载 LCP | ≤2.5s（桌面）/ ≤4.0s（低配） | Lighthouse | Flux |
| 首次输入延迟 FID / INP | ≤200ms | Lighthouse / Chrome DevTools | Flux |
| 分镜节点编辑器 FPS | ≥60 FPS（桌面）/ ≥30 FPS（低配） | Chrome FPS 计数器，200 节点场景 | Flux |
| 节点拖拽/排序响应时间 | ≤200ms | 用户操作到 UI 更新 | Flux |
| 版本历史弹窗打开时间 | ≤500ms（50 个版本） | 点击到首屏渲染 | Flux |
| Diff 对比渲染时间 | ≤300ms（单字段变更） | 从选择版本到高亮完成 | Flux |
| 前端 JS 初始包 gzip | ≤500KB | 构建产物分析 | Flux |

### 2.1 前端优化要求

- 对分镜节点编辑器启用虚拟滚动，单集节点数 >50 时启用。
- 节点缩略图懒加载，使用 `loading="lazy"` 或 Intersection Observer。
- 大 JSON 数据（如完整大纲、脚本）使用 `useMemo` / `useDeferredValue` 避免阻塞输入。
- 避免在 `useEffect` 中直接发起同步大数据计算；使用 Web Worker 处理复杂 diff。

## 3. 后端 API 性能指标

| 指标 | 目标值 | 测量方式 | 责任人 |
|------|--------|----------|--------|
| CRUD API P95 响应时间 | ≤200ms | 生产/压测日志 | Core |
| 列表查询 P95（分页 ≤50 条） | ≤300ms | 生产/压测日志 | Core |
| 版本历史查询（50 版本） | ≤500ms | API 测试 | Core |
| Diff 计算接口 P95 | ≤300ms | API 测试 | Core |
| 一致性检查（剧本医生） | ≤5s（单集/大纲） | API 测试 | Core |
| 变更影响分析 | ≤500ms | API 测试 | Core |
| 数据库连接池等待时间 | ≤50ms P95 | 监控 | Core |
| Redis 命令 P99 | ≤10ms | 监控 | Core |

### 3.1 后端优化要求

- 所有数据库表按 `project_id`、`entity_type`、`entity_id` 建立复合索引。
- 版本快照内容使用 PostgreSQL JSONB，查询时避免全字段反序列化。
- 一致性检查中的规则校验优先在内存中完成，减少数据库往返。
- 大对象（图片/视频元信息）不存入版本快照 diff，只记录引用 ID。

## 4. AI 生成性能指标

| 任务类型 | 目标耗时 | 失败重试 | 降级要求 | 责任人 |
|----------|----------|----------|----------|--------|
| 文本生成（大纲/脚本/台词） | ≤30s | 3 次，指数退避 | 5s 内切换 fallback | Core |
| 图像生成（分镜图/角色/场景） | ≤60s | 3 次 | 5s 内切换 fallback | Core |
| 视频片段生成 | 5-8 分钟/节点 | 2 次 | 5s 内切换 fallback | Core |
| TTS 配音生成 | ≤20s/条 | 3 次 | 5s 内切换 fallback | Core |
| 配乐生成 | ≤60s | 3 次 | 5s 内切换 fallback | Core |
| 合成输出（单集） | ≤5 分钟 | 2 次 | 记录失败原因，人工介入 | Core |

### 4.1 队列与并发要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 队列任务投递成功率 | 100% | 任务创建即入队。 |
| Worker 并发任务数 | ≥100 个并发 AI 任务 | 可通过水平扩展 Worker 提升。 |
| 任务状态同步延迟 | ≤2s | 从 Worker 更新到前端 SSE 收到。 |
| 断点续传 | 支持 | 页面刷新后可通过 job_id 重新订阅 SSE。 |
| 任务失败通知 | ≤5s | 失败后前端立即收到事件。 |

### 4.2 资源使用要求

- 单条 AI 生成 Worker 进程内存占用 ≤2GB（视频生成除外，允许 ≤8GB）。
- FFmpeg 合成任务单进程 CPU 占用不超过 4 核。
- 对象存储上传/下载带宽瓶颈不在应用层（依赖 MinIO/网络）。

## 9. 人类介入红线

修改以下核心指标须人类负责人书面确认（GitHub Issue 记录）：

1. 前端 LCP 目标 >2.5s（桌面）或 >4.0s（低配）。
2. API P95 响应时间 >200ms（CRUD）或 >500ms（版本历史）。
3. 桌面端 FPS 目标 <60 FPS 或低配端 <30 FPS。
4. AI 生成耗时目标放宽超过 20%。
5. 视频分辨率从 1080p 降级为 720p 作为默认。
