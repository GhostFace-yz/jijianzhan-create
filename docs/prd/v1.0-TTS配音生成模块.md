# v1.0 TTS 配音生成模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

短剧需要为每个角色的台词生成配音，传统方式依赖专业配音演员，成本高、周期长。创作者需要按角色绑定音色、根据情绪调整语速的自动化 TTS 配音能力。

### 1.2 解决方案

为每个含台词的节点生成配音音频，自动从角色圣经获取 `voice_id`，根据台词情绪和角色声线配置生成参数。提供试听、修改情绪参数重生、修改台词文本重生、手动上传录音文件等审核操作。

### 1.3 成功标准

- 有台词节点配音生成成功率 ≥ 90%。
- 同一角色全剧使用同一 `voice_id` 执行率 100%。
- 无台词节点自动跳过执行率 100%。
- 配音时长与节点时长匹配误差 ≤ 10%。

## 2. 用户体验与功能

### 2.1 用户画像

- **声音导演**：关注配音情绪、语速和音色一致性。
- **独立创作者**：需要低成本快速生成配音初稿。

### 2.2 用户故事

1. **作为** 声音导演，**我希望** 系统自动为含台词节点生成配音，**以便** 快速获得音频初稿。
2. **作为** 声音导演，**我希望** 试听每句配音，**以便** 判断是否自然。
3. **作为** 声音导演，**我希望** 修改情绪参数后重新生成，**以便** 调整表演效果。
4. **作为** 声音导演，**我希望** 上传自己的录音文件替换 AI 配音，**以便** 使用真人配音。
5. **作为** 独立创作者，**我希望** 无台词节点自动跳过，**以便** 避免无效生成。

### 2.3 验收标准

#### 生成参数

- [ ] 输入参数：`text`（台词）、`voice_id`（角色圣经自动获取）、`emotion`（映射：开心→happy, 悲伤→sad, 愤怒→angry, 平静→neutral, 惊讶→surprised, 紧张→fearful）、`speed`（0.8-1.2，默认 1.0）、`output_format`（wav, 44100Hz, 单声道）。
- [ ] 批量生成：过滤 `dialogue` 不为 null 的节点，后台并行。

#### 情绪与语速映射

- [ ] 情绪标签映射到 TTS 支持的情绪参数。
- [ ] 语速基于角色圣经 `voice.speed` 映射：normal→1.0、fast→1.15、slow→0.85。

#### 审核操作

- [ ] 试听：内嵌播放器播放生成的音频。
- [ ] 修改情绪参数重生：调整 `emotion` 后重新生成。
- [ ] 修改台词文本重生：节点台词修改后重新触发配音生成。
- [ ] 手动上传录音文件：支持 wav/mp3 格式，上传后替换 AI 配音。

#### 与角色圣经联动

- [ ] 生成前自动从角色圣经读取 `voice.voice_id`。
- [ ] 若角色未绑定 `voice_id`，提示用户先完成 TTS 音色绑定。
- [ ] 全剧同一角色强制使用同一 `voice_id`。

#### 状态管理

- [ ] 节点 `audio_clip` 字段存储音频 URL。
- [ ] 配音生成状态与节点状态联动：`pending` → `generating` → `completed`。

### 2.4 非目标

- 本阶段不实现自动对口型（lip-sync）视频生成。
- 本阶段不实现多语言配音同步。
- 本阶段不实现实时变声或音效叠加。

## 3. AI 系统要求

### 3.1 工具要求

- TTS AI Adapter（ElevenLabs、FishSpeech 等）。
- 情绪参数映射表。
- 音频格式转换能力。

### 3.2 评估策略

- 人工试听 30 条配音：自然度评分 ≥ 4/5。
- 自动化检查：音频格式正确率 100%、时长匹配误差 ≤ 10%。
- 情绪识别测试：不同情绪参数生成的音频可被人工正确区分。

## 4. 技术规格

### 4.1 架构概述

```
分镜节点就绪 → 过滤含台词节点 → 读取角色 voice_id → TTSAdapter.generateSpeech() → 返回 audio_clip → 试听/审核
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/episodes/{ep_id}/audio/tts/generate`、`POST /api/v1/projects/{id}/episodes/{ep_id}/audio/tts/nodes/{node_id}/generate`、`PUT /api/v1/projects/{id}/episodes/{ep_id}/audio/tts/nodes/{node_id}/review`。
- **数据库**: `storyboard_nodes` 表的 `audio_clip` 字段。
- **对象存储**: MinIO/S3 存储音频文件。
- **AI 适配器层**: `TTSAdapterPool`。

### 4.3 安全与隐私

- 上传的录音文件进行内容安全审核。
- 音频数据按项目隔离。
- TTS 克隆音色不用于模型训练。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 单节点/批量配音生成、试听、情绪参数调整。
- **v1.1**: 手动上传录音、角色 voice_id 缺失提示。
- **v2.0**: 自动对口型参考、多语言配音、音效库集成。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| TTS 情绪表达不自然 | 中 | 提供多种情绪参数试听，允许手动上传 |
| 长台词生成失败 | 中 | 自动切分长文本，合并音频 |
| 角色 voice_id 未绑定 | 中 | 生成前强制校验并提示 |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成、角色圣经、单集脚本生成、分镜节点化。
- 被依赖模块：视频片段生成、合成输出。

## 7. 验收测试要点

1. 为一集 10 个节点生成配音，验证无台词节点被跳过。
2. 试听生成的配音，验证情绪表达自然。
3. 修改台词后重新生成配音，验证 `audio_clip` 更新。
4. 上传录音文件替换 AI 配音，验证存储和播放。
5. 验证同一角色在不同节点使用同一 `voice_id`。
6. 测试长台词（>200 字）生成，验证自动切分或成功生成。
