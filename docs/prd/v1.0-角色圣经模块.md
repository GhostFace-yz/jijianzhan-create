# v1.0 角色圣经模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

短剧角色需要在整剧中保持一致的外貌、服装、表情和声音。传统方式依赖人工整理参考图和文档，容易遗漏细节，导致不同镜头中角色形象不一致。

### 1.2 解决方案

根据已确认的大纲 `characters[]` 自动创建角色卡片，提供外貌、服装、表情、声线、参考图集的管理能力。通过 IP-Adapter 参考图注入、三视图生成、表情组/场景站立图扩展，确保后续图像和视频生成中角色形象一致。

### 1.3 成功标准

- 角色卡片创建成功率 100%（基于大纲数据）。
- 三视图生成成功率 ≥ 85%。
- 参考图集入库后，后续涉及该角色的图像生成注入率 100%。
- 用户对角色字段编辑支持版本快照，回滚成功率 100%。

## 2. 用户体验与功能

### 2.1 用户画像

- **视觉导向创作者**：重视角色外观一致性，需要参考图辅助决策。
- **声音敏感创作者**：需要为角色匹配合适的 TTS 音色或克隆声音。

### 2.2 用户故事

1. **作为** 创作者，**我希望** 根据大纲自动创建角色卡片，**以便** 快速开始角色设计。
2. **作为** 创作者，**我希望** 为每个角色生成三视图，**以便** 获得多角度的视觉参考。
3. **作为** 创作者，**我希望** 生成角色的表情组和场景站立图，**以便** 构建完整的参考图集。
4. **作为** 创作者，**我希望** 为角色添加多套服装变体，**以便** 在分镜中选择不同造型。
5. **作为** 创作者，**我希望** 为角色绑定 TTS 音色，**以便** 后续配音保持同一声音。
6. **作为** 创作者，**我希望** 查看和回滚角色的历史版本，**以便** 撤销不满意修改。

### 2.3 验收标准

#### 角色卡片基础字段

- [ ] 自动从大纲 `characters[]` 创建角色卡片。
- [ ] 字段包括：姓名、角色定位（主角/配角/反派）、登场集数范围、外貌描述（发型/面部/身高/肤色/年龄）、服装方案、表情特征、标志性动作、声线描述。
- [ ] 所有字段支持编辑，编辑后触发版本快照。
- [ ] 角色状态：`draft` → `confirmed`（确认后才可在分镜中使用）。

#### 三视图生成

- [ ] 用户点击「生成三视图」后，系统组装 Prompt（正面/侧面/背面）。
- [ ] 固定负向 Prompt：`multiple people, extra limbs, bad anatomy, blurry, watermark, text, logo, nsfw`。
- [ ] 三张图同时生成，并排展示。
- [ ] 可对单张图「重新生成」（不影响其他两张）。
- [ ] 可「修改描述」后重生成。
- [ ] 三张图全部满意后点击「确认三视图」。

#### 参考图集构建

- [ ] 确认三视图后自动触发扩展图生成：表情组 4 张 + 场景站立图 2 张，共 9 张入库。
- [ ] 9 张图上传至图像 AI 的 IP-Adapter 接口，获取角色 Embedding。
- [ ] 存储：`ref_images[]`、`ip_adapter_id`、`lora_path`。
- [ ] 后续所有涉及该角色的图像生成均注入此权重，IP-Adapter 权重建议 0.7-0.85。

#### 服装变体管理

- [ ] 每套服装方案旁提供「生成变体图」按钮。
- [ ] 生成全身图，确认后存入 `costumes` 字典。
- [ ] 分镜节点中通过下拉菜单选择服装变体。
- [ ] 生成时叠加注入对应变体图，优先级高于主参考图集。

#### 多角色场景一致性

- [ ] 每个角色的 IP-Adapter 参考图分别注入，权重独立。
- [ ] 多角色时建议权重从 0.8 降至 0.65。
- [ ] 超过 3 个角色同框时系统提示一致性风险，建议使用远景或群像镜头。
- [ ] 支持用户手动上传多人同框参考图。

#### TTS 音色绑定

- [ ] 方案 A：预置库按性别/年龄/风格筛选，试听确认。
- [ ] 方案 B：上传 ≥ 30 秒参考音频克隆，测试验证后绑定 `voice_id`。
- [ ] 全剧同一角色使用同一 `voice_id`。

#### 版本快照

- [ ] 所有字段支持版本快照，可回滚到任意历史版本。
- [ ] 每次 AI 生成、用户编辑、重新生成均创建新版本。

### 2.4 非目标

- 本阶段不实现角色的 3D 模型绑定。
- 本阶段不实现角色动作捕捉。
- 本阶段不实现角色在商业授权层面的 IP 保护机制。

## 3. AI 系统要求

### 3.1 工具要求

- 图像 AI Adapter（Flux、NanoBanana、SDXL 等）。
- IP-Adapter / ControlNet 参考图注入能力。
- TTS Adapter（ElevenLabs、FishSpeech 等）。

### 3.2 评估策略

- 人工评估三视图一致性（同一角色三张图相似度评分 ≥ 4/5）。
- 自动化评估：IP-Adapter 注入后生成图像的角色一致性评分 ≥ 0.85。
- TTS 音色绑定后试听满意度 ≥ 80%。

## 4. 技术规格

### 4.1 架构概述

```
大纲确认 → 自动创建角色卡片 → 用户编辑/生成三视图 → 扩展图生成 → IP-Adapter Embedding → 存储引用 → 分镜节点使用
```

### 4.2 集成点

- **API**: `GET/PUT /api/v1/projects/{id}/characters`、`POST /api/v1/projects/{id}/characters/{char_id}/generate-views`、`POST /api/v1/projects/{id}/characters/{char_id}/generate-refs`、`POST /api/v1/projects/{id}/characters/{char_id}/bind-voice`。
- **数据库**: `characters` 表或项目 JSONB 中的 `asset_library.characters`。
- **对象存储**: MinIO/S3 存储参考图。
- **AI 适配器层**: `ImageAdapterPool`、`TTSAdapterPool`。

### 4.3 安全与隐私

- 用户上传的参考音频和图像需进行内容安全审核。
- 角色数据属于项目资产，按项目隔离。
- TTS 克隆音频不用于模型训练。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 自动创建角色卡片、基础字段编辑、三视图生成、参考图集构建。
- **v1.1**: 服装变体、TTS 音色绑定、多人同框风险提示。
- **v2.0**: 角色 Embedding 自动优化、跨项目角色复用、角色表情视频预览。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 三视图角色不一致 | 高 | 固定 Seed 范围 + 强角色描述 + 负向 Prompt |
| IP-Adapter 权重不当导致崩坏 | 中 | 提供权重滑块默认值，允许用户微调 |
| TTS 克隆音频质量差 | 中 | 限制音频格式（≥30s、清晰、单声道） |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成。
- 被依赖模块：单集脚本生成、分镜节点化、分镜图生成、TTS 配音生成、视频片段生成。

## 7. 验收测试要点

1. 基于测试大纲生成 3 个角色，验证卡片字段完整性。
2. 为一个角色生成三视图，验证三张图并排展示和单张重生成。
3. 确认三视图后验证是否自动触发 6 张扩展图生成并入库。
4. 添加 2 套服装变体，验证分镜节点选择和生成注入。
5. 绑定 TTS 音色并试听，验证同一角色后续配音使用同一 voice_id。
6. 修改角色外貌描述，验证版本快照生成和回滚。
