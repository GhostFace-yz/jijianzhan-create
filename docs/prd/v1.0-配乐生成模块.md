# v1.0 配乐生成模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

背景音乐对短剧情绪节奏至关重要。手动挑选或创作 BGM 耗时，且难以与分镜节点的时间轴和情绪变化对齐。创作者需要 AI 生成与情绪序列匹配的整集配乐，并自动切割对齐到节点。

### 1.2 解决方案

以整集为单位生成背景音乐，从所有节点的 `music_mood` 提取情绪序列，结合项目题材和时长目标生成配乐。生成后支持试听、重新生成或手动上传，并按节点时长切割对齐，进行 Duck 处理。

### 1.3 成功标准

- 整集配乐生成成功率 ≥ 80%。
- 配乐按节点时长切割对齐误差 ≤ 0.5 秒。
- 有台词节点配乐压低至 20-30%，无台词节点保持 80-100%。
- 相邻节点情绪差异过大时提示插入过渡节点，提示覆盖率 100%。

## 2. 用户体验与功能

### 2.1 用户画像

- **音乐总监**：关注配乐情绪起伏和与画面的同步。
- **独立创作者**：需要快速生成可使用的 BGM，降低音乐成本。

### 2.2 用户故事

1. **作为** 音乐总监，**我希望** 根据全集情绪序列生成整段 BGM，**以便** 获得与剧情匹配的音乐。
2. **作为** 音乐总监，**我希望** 试听整段 BGM，**以便** 判断是否满意。
3. **作为** 音乐总监，**我希望** BGM 按节点时长自动切割并对齐，**以便** 后续合成使用。
4. **作为** 音乐总监，**我希望** 有台词时配乐自动压低，**以便** 保证台词清晰。
5. **作为** 独立创作者，**我希望** 手动上传自己的 BGM，**以便** 使用已有音乐素材。

### 2.3 验收标准

#### 情绪序列提取

- [ ] 从所有节点的 `music_mood` 字段提取情绪序列，如「轻松→紧张→悬疑→温馨→激昂」。
- [ ] 统计每个情绪段的节点数量和累计时长。

#### 生成参数

- [ ] `style_tags`：由项目题材驱动。
- [ ] `emotion_sequence`：节点情绪序列。
- [ ] `duration`：节点数 × 6.5 秒估算。
- [ ] `instrument_pref`：用户可选乐器偏好（可选）。

#### 生成与试听

- [ ] 整集配乐生成后提供试听播放器。
- [ ] 不满意可重新生成或手动上传 BGM。
- [ ] 手动上传支持 mp3/wav 格式。

#### 配乐切割与对齐

- [ ] 按各节点 `duration_target` 累计时间点切割为 N 段。
- [ ] 相邻片段间 0.3-0.5 秒交叉淡化。
- [ ] 相邻节点 `music_mood` 差异过大时提示插入过渡节点。

#### Duck 处理

- [ ] 有台词节点配乐压低至 20-30%（默认 30%）。
- [ ] 无台词节点配乐保持 80-100%（默认 80%）。
- [ ] 音量变化渐变 0.2 秒。

#### 状态管理

- [ ] 整集配乐 URL 存储在 `Episode.music.original_url`。
- [ ] 切割后的片段存储在 `Episode.music.segments`。

### 2.4 非目标

- 本阶段不实现根据画面内容实时生成音乐。
- 本阶段不实现多音轨混音（除 Duck 处理外）。
- 本阶段不实现版权音乐库集成。

## 3. AI 系统要求

### 3.1 工具要求

- 音乐 AI Adapter（Suno、Udio 等）。
- 音频切割和淡入淡出处理能力。
- 情绪序列解析。

### 3.2 评估策略

- 人工试听 10 集配乐：情绪匹配度 ≥ 4/5。
- 自动化检查：切割点时间误差 ≤ 0.5 秒、Duck 处理后音量符合范围。
- 生成失败率统计：目标 ≤ 20%。

## 4. 技术规格

### 4.1 架构概述

```
分镜节点就绪 → 提取 music_mood 序列 → MusicAdapter.generateMusic() → 返回整段 BGM → 按节点时长切割 → Duck 处理 → 存储 segments
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/episodes/{ep_id}/audio/music/generate`、`PUT /api/v1/projects/{id}/episodes/{ep_id}/audio/music/upload`。
- **数据库**: `episodes` 表的 `music` 字段（JSONB）。
- **对象存储**: MinIO/S3 存储音乐文件。
- **AI 适配器层**: `MusicAdapterPool`。
- **音频处理**: FFmpeg 或音频处理库。

### 4.3 安全与隐私

- 上传的 BGM 进行版权和敏感内容审核（当前阶段为形式审核）。
- 音乐数据按项目隔离。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 情绪序列提取、整集配乐生成、试听、手动上传。
- **v1.1**: 自动切割对齐、Duck 处理、过渡节点提示。
- **v2.0**: 智能乐器选择、版权音乐库、多音轨混音。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 音乐生成时长与目标不符 | 中 | 允许 ±5 秒偏差，FFmpeg 裁剪或循环填充 |
| 情绪切换突兀 | 中 | 提示插入过渡节点 + 交叉淡化 |
| 音乐生成成本高昂 | 中 | 提供手动上传作为降级方案 |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成、单集脚本生成、分镜节点化。
- 被依赖模块：合成输出。

## 7. 验收测试要点

1. 为一集 10 个节点生成配乐，验证情绪序列提取正确。
2. 试听整段 BGM，验证可播放。
3. 验证切割后的片段数量等于节点数量。
4. 验证相邻片段间交叉淡化时长在 0.3-0.5 秒。
5. 测试有台词和无台词节点的 Duck 处理音量。
6. 手动上传 BGM，验证替换成功并按节点切割。
