# v1.0 合成输出模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

所有视频片段、配音和配乐准备就绪后，需要将它们拼接、混音、加字幕，输出最终 MP4。手动使用专业剪辑软件门槛高，创作者需要自动化的合成输出能力。

### 1.2 解决方案

用户点击「开始合成」后，系统将所有片段按时间轴拼接，根据相邻节点关系应用转场，混合配音和配乐，添加字幕，最终输出 MP4。提供实时进度、预览、下载和重新合成能力。

### 1.3 成功标准

- 合成输出成功率 ≥ 90%。
- 合成时长 ≤ 5 分钟/集（20-30 节点）。
- 最终视频时长与目标时长误差 ≤ 5%。
- 字幕与配音对齐误差 ≤ 0.3 秒。

## 2. 用户体验与功能

### 2.1 用户画像

- **导演/制片人**：需要快速输出成片并下载。
- **独立创作者**：需要简单配置即可获得可用视频。

### 2.2 用户故事

1. **作为** 导演，**我希望** 点击开始合成后自动拼接所有片段，**以便** 获得最终视频。
2. **作为** 导演，**我希望** 看到实时合成进度，**以便** 了解当前阶段。
3. **作为** 导演，**我希望** 预览合成的最终视频，**以便** 确认效果。
4. **作为** 导演，**我希望** 下载高清 MP4，**以便** 分发到各大平台。
5. **作为** 独立创作者，**我希望** 返回具体节点修改后重新合成，**以便** 快速迭代。

### 2.3 验收标准

#### 合成参数配置

- [ ] 分辨率：1080×1920（竖屏默认）/ 1920×1080（横屏）。
- [ ] 帧率：30fps（默认）/ 24fps。
- [ ] 编码：H.264（默认）/ H.265。
- [ ] 字幕：开启/关闭、来源（台词自动对齐）、位置（底部/顶部）、样式（白字黑边/纯白/纯黑）、大小（小/中/大）。
- [ ] 音频混音：配音 100%、有台词段配乐 20-40%（默认 30%）、无台词段配乐 60-90%（默认 80%）。

#### 转场处理

- [ ] 同场景相邻节点：直接拼接。
- [ ] 同场景不同时间段：0.3 秒叠化（Dissolve）。
- [ ] 不同场景之间：匹配帧过渡（取上节点末 3 帧与下节点前 3 帧相似度比较，高则直拼，低则 0.5 秒淡入淡出）。
- [ ] 情绪强转场（闪回/梦境/时间跳跃）：用户手动指定「白场闪」（0.1s 白场+直切）或「黑场」（0.3s 淡出至黑+淡入）。

#### 时长对齐

- [ ] 配音超过视频时长：视频末帧冻结延伸，确保配音播完。
- [ ] 配音短于视频：保持视频时长，配音结束后画面继续。

#### 合成流程

- [ ] 用户点击「开始合成」，进入渲染队列（预计 3-8 分钟）。
- [ ] 实时进度条：拼接中 → 混音中 → 字幕渲染中 → 编码中。
- [ ] 完成后：内嵌播放器预览、高清下载链接、状态更新为「已完成」。
- [ ] 可返回具体节点修改后「重新合成」。

#### 字幕生成

- [ ] 从节点 `dialogue.text` 自动提取字幕文本。
- [ ] 根据配音音频时长对齐字幕时间轴。
- [ ] 支持字幕样式配置。

### 2.4 非目标

- 本阶段不实现复杂的非线性剪辑（如多轨道、画中画）。
- 本阶段不实现 AI 自动调色。
- 本阶段不实现多平台格式自动适配（除竖屏/横屏外）。

## 3. AI 系统要求

本模块不涉及 AI 生成，主要依赖 FFmpeg 等媒体处理工具。

## 4. 技术规格

### 4.1 架构概述

```
所有节点视频通过 → RenderService 按时间轴拼接 → 转场 → 混音（配音+配乐 Duck）→ 字幕渲染 → FFmpeg 编码 → 输出 MP4
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/episodes/{ep_id}/render`、`GET /api/v1/projects/{id}/episodes/{ep_id}/render/progress`、`GET /api/v1/projects/{id}/episodes/{ep_id}/render/download`。
- **数据库**: `episodes` 表的 `output_video`、`status` 字段。
- **对象存储**: MinIO/S3 存储最终视频。
- **媒体处理**: FFmpeg。
- **任务队列**: BullMQ + Redis。

### 4.3 安全与隐私

- 最终视频按项目隔离存储。
- 下载链接带有时效性或权限校验（当前阶段可公开但按项目隔离）。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 片段拼接、基础转场、混音、字幕、MP4 输出。
- **v1.1**: 匹配帧过渡、情绪强转场、参数配置。
- **v2.0**: 多轨道混音、自动调色、多平台格式导出。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| FFmpeg 编码失败 | 高 | 详细的错误日志 + 分段重试 |
| 最终视频时长偏差大 | 中 | 末帧冻结或保留视频时长 |
| 大文件传输超时 | 中 | 分片上传/下载、CDN 分发 |

## 6. 依赖关系

- 前置依赖：项目创建、单集脚本生成、分镜节点化、分镜图生成、TTS 配音生成、配乐生成、视频片段生成。
- 被依赖模块：无。

## 7. 验收测试要点

1. 为一集 10 个节点执行合成，验证最终 MP4 可播放。
2. 验证字幕与配音对齐误差 ≤ 0.3 秒。
3. 测试不同场景间转场效果（淡入淡出/叠化）。
4. 测试配音超过视频时长的情况，验证末帧冻结。
5. 测试重新合成：修改一个节点后重新执行合成。
6. 测试下载链接可正常下载最终视频。
