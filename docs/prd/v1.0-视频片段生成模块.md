# v1.0 视频片段生成模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

分镜图和配音准备就绪后，需要将它们转化为 5-8 秒的视频片段。视频生成成本高、耗时长，且容易出现人脸崩坏、动作跳变等问题。创作者需要可靠的批量视频生成、质量检测和降级熔断机制。

### 1.2 解决方案

为每个通过分镜图审核的节点生成视频片段，输入首帧图（分镜图）、角色参考图、场景参考图、运镜指令和运动描述。提供批量生成、质量自动检测、降级熔断、片段预览和重生成能力。

### 1.3 成功标准

- 单节点视频生成成功率 ≥ 80%。
- 生成视频时长与目标时长误差 ≤ 1 秒。
- 人脸崩坏/动作跳变检测覆盖率 100%。
- 首选模型失败后自动降级到 fallback 模型的成功率 ≥ 90%。

## 2. 用户体验与功能

### 2.1 用户画像

- **导演**：需要预览每个镜头并控制生成质量。
- **后期制作**：需要可复用的视频片段进入最终合成。

### 2.2 用户故事

1. **作为** 导演，**我希望** 为单个节点生成视频片段，**以便** 看到动态效果。
2. **作为** 导演，**我希望** 批量生成所有通过审核的节点视频，**以便** 提高效率。
3. **作为** 导演，**我希望** 预览生成的视频片段，**以便** 判断质量是否通过。
4. **作为** 导演，**我希望** 对不满意的片段重新生成或调整参数，**以便** 获得更好的效果。
5. **作为** 后期制作，**我希望** 所有片段通过后才解锁合成，**以便** 避免使用低质量素材。

### 2.3 验收标准

#### 输入清单

- [ ] 首帧图像：分镜图。
- [ ] 角色参考图：IP-Adapter 参考图。
- [ ] 场景参考图：场景基准图/变体图。
- [ ] 运镜指令：`camera_move` 映射后的平台参数。
- [ ] 运动描述：从 `visual_desc` 和镜头类型推导。
- [ ] 配音音频：可选，用于口型参考。

#### 运镜参数映射

- [ ] `static` → `none`。
- [ ] `zoom_in` → `zoom in`。
- [ ] `zoom_out` → `zoom out`。
- [ ] `pan_left` → `pan left`。
- [ ] `pan_right` → `pan right`。
- [ ] `follow` → `tracking`。
- [ ] `crane_up` → `ascend`。
- [ ] `crane_down` → `descend`。
- [ ] 用户在节点选择通用描述，系统自动转换为对应平台参数。

#### 生成策略

- [ ] 主模式：图生视频（分镜图为首帧）。
- [ ] 人脸增强：特写/近景含人脸节点开启 Face Enhancement。
- [ ] 批量生成：加入队列后台并行，用户可离开页面，完成后通知。

#### 质量自动检测

- [ ] 时长检测：实际时长与目标时长误差 ≤ 1 秒。
- [ ] 人脸崩坏检测：特写节点进行人脸质量评估。
- [ ] 动作跳变/闪烁检测：帧间差异分析。
- [ ] 检测不通过自动标记「需重做」，展示检测报告。

#### 降级熔断机制

- [ ] 首选模型失败（Rate Limit / Timeout）→ 自动切换 `video_fallback` → 系统默认视频模型。
- [ ] 非可用性错误直接抛给用户。

#### 片段预览与重生成

- [ ] 内嵌播放，支持连续播放相邻节点检查衔接。
- [ ] 操作：通过 / 重新生成（随机 Seed）/ 调整参数重生（运镜强度、运动描述、首帧替换）。
- [ ] 所有节点通过后解锁「合成输出」。

### 2.4 非目标

- 本阶段不实现实时视频渲染预览。
- 本阶段不实现视频风格迁移（统一风格在分镜图阶段完成）。
- 本阶段不实现多镜头复杂剪辑。

## 3. AI 系统要求

### 3.1 工具要求

- 视频 AI Adapter（Kling、Sora、Runway 等）。
- 图生视频能力。
- 人脸质量检测模型（可选）。
- 帧间差异分析能力。

### 3.2 评估策略

- 人工评估 20 个视频片段：画面稳定性、运镜准确性、人脸一致性。
- 自动化检测：时长误差、人脸崩坏率、闪烁跳变率。
- 降级熔断成功率统计：目标 ≥ 90%。

## 4. 技术规格

### 4.1 架构概述

```
分镜图审核通过 → 组装视频生成参数 → VideoAdapter.generateVideo() → 质量检测 → 通过/重做/降级
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/episodes/{ep_id}/video/generate`、`POST /api/v1/projects/{id}/episodes/{ep_id}/video/nodes/{node_id}/generate`、`PUT /api/v1/projects/{id}/episodes/{ep_id}/video/nodes/{node_id}/review`。
- **数据库**: `storyboard_nodes` 表的 `video_clip`、`status`、`ai_model_used` 字段。
- **对象存储**: MinIO/S3 存储视频片段。
- **AI 适配器层**: `VideoAdapterPool`。
- **任务队列**: BullMQ + Redis。

### 4.3 安全与隐私

- 视频生成参数按项目隔离。
- 生成视频进行内容安全审核。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 单节点/批量视频生成、基础预览、通过/重做。
- **v1.1**: 人脸增强、质量自动检测、降级熔断。
- **v2.0**: 智能生成参数优化、视频风格一致性增强、口型参考。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 视频生成成本高、耗时长 | 高 | 后台异步队列、降级熔断、草稿模式 |
| 人脸崩坏导致可用率低 | 高 | Face Enhancement + 人脸检测 + 自动重做 |
| 模型 API 不稳定 | 中 | 多模型 fallback + 自动重试 |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成、角色圣经、场景圣经、单集脚本生成、分镜节点化、分镜图生成、TTS 配音生成（可选）。
- 被依赖模块：合成输出。

## 7. 验收测试要点

1. 为 5 个节点生成视频，验证视频 URL 存储和时长。
2. 测试运镜参数映射：每个 camera_move 正确转换。
3. 构造人脸特写节点，验证 Face Enhancement 开启。
4. 构造质量不合格视频，验证自动标记「需重做」。
5. 模拟首选模型失败，验证降级到 fallback 模型。
6. 测试批量生成：所有通过分镜图审核的节点进入队列。
