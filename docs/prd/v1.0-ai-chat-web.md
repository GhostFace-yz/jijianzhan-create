## 1. 执行摘要

### 1.1 产品定位
为 C 端消费者提供一款基于浏览器的 AI 对话产品，支持日常文本对话（kimi）与图片/视频理解（agnes-ai）两种模型能力，通过服务端统一中转实现安全、可控的多模态交互体验。

### 1.2 核心目标
- **MVP 目标**：用户在注册/登录后，可通过网页端与 AI 进行文本对话或上传图片/视频获得理解分析，且对话历史在多设备间自动同步。
- **成功指标**：首屏加载 ≤ 2s；流式响应首字延迟 ≤ 500ms；支持最新 2 个主版本的 Chrome/Edge/Safari/Firefox。

### 1.3 目标用户
| 维度 | 描述 |
|------|------|
| 年龄层 | 18–45 岁 |
| 技术素养 | 具备基础互联网使用能力，对 AI 对话工具有初步认知 |
| 使用场景 | 日常问答、学习辅助、内容创作、图片/视频理解 |
| 设备 | 桌面浏览器为主，移动端浏览器为辅 |

## 2. 用户故事

### US-1 注册与登录
**作为** 新用户，**我希望** 通过手机号或邮箱注册并登录，**以便** 保存我的对话历史并在多设备同步。

**验收标准（AC）**：
- [ ] 支持手机号+验证码注册，验证码 60s 内有效
- [ ] 支持邮箱+密码注册，密码强度要求 ≥ 8 位，含大小写字母+数字
- [ ] 登录后返回 JWT Token，有效期 7 天，支持自动续期
- [ ] 未登录用户访问对话页时，强制跳转登录页
- [ ] 支持密码找回（手机号/邮箱验证后重置）

### US-2 发起文本对话
**作为** 已登录用户，**我希望** 在输入框中输入文本并发送，**以便** 获得 AI 的即时回复。

**验收标准（AC）**：
- [ ] 输入框支持多行文本，Enter 发送，Shift+Enter 换行
- [ ] 单次输入上限 4000 字符（含换行），超限给出明确提示
- [ ] 发送后消息立即出现在对话区（本地乐观更新）
- [ ] AI 回复以 SSE 流式方式逐字展示，延迟 ≤ 500ms（首字）
- [ ] 回复内容支持 Markdown 渲染（标题、列表、加粗、链接、引用）
- [ ] 代码块支持语法高亮，并提供「一键复制」按钮

### US-3 上传图片/视频进行理解
**作为** 已登录用户，**我希望** 在对话中上传图片或视频，**以便** AI 基于视觉内容进行分析回答。

**验收标准（AC）**：
- [ ] 支持上传格式：图片 JPG/PNG/GIF/WEBP（单文件 ≤ 10MB）；视频 MP4/MOV（单文件 ≤ 50MB）
- [ ] 上传前进行格式与大小校验，失败时给出明确错误提示
- [ ] 图片/视频在消息中可预览，支持点击放大查看
- [ ] 含图片/视频的消息由服务端自动路由至 agnes-ai 处理
- [ ] 纯文本消息由服务端自动路由至 kimi 处理
- [ ] 上传进度可视化（百分比或进度条）

### US-4 管理会话历史
**作为** 已登录用户，**我希望** 查看、继续和删除历史会话，**以便** 高效管理我的对话记录。

**验收标准（AC）**：
- [ ] 侧边栏展示最近 50 条会话，按最后活跃时间倒序排列
- [ ] 每条会话显示自动生成的标题（基于首条用户消息摘要，≤ 20 字符）和最后活跃时间
- [ ] 点击历史会话可加载完整对话上下文并继续对话
- [ ] 支持删除单条会话，删除后从列表移除并软删除（服务端保留 30 天可恢复）
- [ ] 支持「新建对话」按钮，清空当前上下文开始新会话

### US-5 多设备同步
**作为** 已登录用户，**我希望** 在设备 A 发起的对话在设备 B 登录后可见，**以便** 无缝切换设备使用。

**验收标准（AC）**：
- [ ] 所有会话和消息实时持久化至服务端数据库
- [ ] 换设备登录后，历史会话列表完整同步（≤ 3s 加载完成）
- [ ] 同一账号多设备同时在线时，新消息在各设备实时同步（WebSocket/SSE 推送）
- [ ] 离线期间产生的对话，在重新联网后自动同步至服务端

### US-6 消息操作
**作为** 已登录用户，**我希望** 对 AI 回复进行重新生成、复制和停止操作，**以便** 更好地控制对话体验。

**验收标准（AC）**：
- [ ] 每条 AI 消息旁提供「重新生成」按钮，点击后基于相同上下文重新请求
- [ ] 提供「复制」按钮，点击后将消息纯文本复制到剪贴板
- [ ] 流式生成过程中显示「停止生成」按钮，点击后立即中断响应
- [ ] 重新生成后保留原消息历史（支持查看/切换历史版本）

### US-7 预设 Prompt / 角色模板（P2）
**作为** 已登录用户，**我希望** 选择预设的角色模板（如「编程助手」「翻译专家」），**以便** 快速切换对话风格。

**验收标准（AC）**：
- [ ] 系统预置 ≥ 5 个角色模板，每个模板含角色描述和系统 Prompt
- [ ] 新建对话时可选择角色模板，选择后该会话所有消息基于对应 Prompt 上下文
- [ ] 支持用户自定义并保存个人角色模板（上限 10 个）

## 3. 功能说明

### 3.1 模型路由策略
服务端根据消息内容自动选择模型供应商，路由规则如下：

| 消息类型 | 路由目标 | 说明 |
|----------|----------|------|
| 纯文本 | kimi | 日常对话、问答、创作 |
| 含图片 | agnes-ai | 图片理解、分析、描述 |
| 含视频 | agnes-ai | 视频内容理解、分析 |
| 文本+图片混合 | agnes-ai | 多模态理解优先 |

### 3.2 会话状态机

```
[新建] → [进行中] → [已归档]
   ↓         ↓
[已删除]  [等待回复]
```

- **新建**：用户点击「新建对话」，会话创建但无消息
- **进行中**：至少有一条消息，且最后一条为 AI 回复
- **等待回复**：用户发送消息后，AI 正在生成回复
- **已归档**：用户主动归档（P2 功能），不再显示在默认列表
- **已删除**：用户删除，服务端软删除保留 30 天

### 3.3 数据量级上限

| 数据项 | MVP 上限 | 说明 |
|--------|----------|------|
| 单用户会话数 | 无上限 | 需分页加载 |
| 单会话消息数 | 200 条 | 超出时提示新建会话 |
| 单条消息文本长度 | 4000 字符 | 输入框硬性限制 |
| 单张图片 | 10 MB | 格式：JPG/PNG/GIF/WEBP |
| 单个视频 | 50 MB | 格式：MP4/MOV，时长 ≤ 5 分钟 |
| 上下文 Token 数 | 🟡 待确认 | 取决于 kimi/agnes-ai 的上下文窗口限制 |

## 4. 非功能需求

### 4.1 性能指标

| 指标 | 目标值 | 验收方式 |
|------|--------|----------|
| 首屏加载时间（FCP） | ≤ 2s | Lighthouse 测量 |
| 首字节时间（TTFB） | ≤ 800ms | 服务端日志 |
| 流式响应首字延迟 | ≤ 500ms | 客户端埋点 |
| 历史会话列表加载 | ≤ 3s（50 条） | 客户端埋点 |
| 桌面端渲染帧率 | ≥ 60 FPS | Chrome DevTools Performance |
| 低配设备渲染帧率 | ≥ 30 FPS | Chrome DevTools Performance |
| 消息列表滚动 | 无掉帧、无白屏 | 人工测试 |

### 4.2 兼容性

| 平台 | 支持范围 |
|------|----------|
| Chrome | 最新 2 个主版本 |
| Edge | 最新 2 个主版本 |
| Safari | 最新 2 个主版本 |
| Firefox | 最新 2 个主版本 |
| 移动端浏览器 | iOS Safari / Chrome Android 最新 2 个主版本 |

### 4.3 响应式断点

| 断点 | 布局 |
|------|------|
| ≥ 1024px（桌面端） | 侧边栏（280px）+ 主对话区（自适应） |
| 768px–1023px（平板） | 侧边栏可收起，主对话区全宽 |
| < 768px（移动端） | 侧边栏全屏抽屉式，主对话区全宽，底部固定输入框 |

### 4.4 安全需求

> 以下安全验收标准由人类负责人在 PRD 评审时注入，Anchor 负责写入，Core/Flux 在实现时须满足。

- **API Key 安全**：模型供应商 API Key 仅存储于服务端，禁止暴露至前端；存储时加密（AES-256）
- **用户鉴权**：JWT Secret 定期轮换（建议 30 天）；Token 泄露后支持服务端强制注销
- **内容审核**：🟡 待确认 — 建议接入敏感内容过滤（文本+图片），违规内容拒绝响应并记录审计日志
- **限流防刷**：单用户每分钟请求上限 30 次（文本）/ 10 次（多媒体），超限返回 429
- **数据隐私**：用户对话内容持久化需符合《个人信息保护法》；服务端日志脱敏处理
- **传输安全**：全站 HTTPS，HSTS 启用，Cookie Secure + HttpOnly + SameSite=Strict

## 5. 排除项（Won't Have）

以下功能明确不在 MVP v1.0 范围内：

1. **文件上传 RAG 问答**：不支持上传 PDF/Word/TXT 并基于文档内容问答（P2 评估）
2. **语音输入/输出**：不支持语音转文字或文字转语音
3. **实时协作**：不支持多人同时参与同一会话
4. **商业化体系**：不支持付费订阅、会员等级、额度购买（待商业模式确认后 P2 评估）
5. **开放 API / 第三方集成**：不提供对外 API 或 Webhook
6. **插件/扩展市场**：不支持用户安装第三方插件

## 6. 风险分析

| 风险项 | 影响 | 概率 | 缓解措施 |
|--------|------|------|----------|
| agnes-ai API 能力边界不明确 | 高 | 中 | 尽快获取 API 文档，明确支持格式、大小限制、响应结构 |
| 流式响应在高并发下延迟抖动 | 高 | 中 | 服务端引入连接池和队列机制，设置超时兜底 |
| 多媒体上传带宽/存储成本超预期 | 中 | 中 | MVP 限制单文件大小，后续评估对象存储+CDN 方案 |
| 内容审核机制缺失导致合规风险 | 高 | 中 | Shield 测试阶段覆盖敏感词场景，建议 P1 接入基础审核 |
| 同一账号多设备同时在线状态冲突 | 中 | 低 | WebSocket 连接管理，服务端维护设备在线状态 |
