# v1.0 场景圣经模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

短剧场景需要在不同镜头中保持空间、风格、色调和光线的一致性。手工管理场景描述和参考图容易遗漏细节，导致同一空间在不同镜头中看起来不同。

### 1.2 解决方案

根据已确认的大纲 `locations[]` 自动创建场景卡片，提供场景描述、变体配置、基准图生成、Prompt 分层管理等功能。通过锁定 base_seed 和变体图预生成，确保同场景所有图像生成使用一致的视觉基础。

### 1.3 成功标准

- 场景卡片创建成功率 100%（基于大纲数据）。
- 同场景基准图生成后，后续所有变体/节点生成强制使用相同 Seed，执行率 100%。
- 用户对场景 Prompt 分层编辑后，生成结果可预览，预览可用率 ≥ 95%。
- 场景变体（日间/夜晚/雨天等）生成成功率 ≥ 85%。

## 2. 用户体验与功能

### 2.1 用户画像

- **视觉导演**：关注场景氛围和光线，需要精细控制画面风格。
- **美术指导**：需要管理场景变体，确保不同时间段/天气下场景一致性。

### 2.2 用户故事

1. **作为** 美术指导，**我希望** 根据大纲自动创建场景卡片，**以便** 快速建立场景资产库。
2. **作为** 美术指导，**我希望** 为每个场景生成基准图并锁定 Seed，**以便** 后续生成保持空间一致。
3. **作为** 美术指导，**我希望** 生成不同时间段和天气的变体图，**以便** 分镜节点按需求选用。
4. **作为** 视觉导演，**我希望** 分层编辑 Prompt（环境/光线/风格/细节/负向），**以便** 精细控制画面效果。
5. **作为** 视觉导演，**我希望** 查看场景的历史版本，**以便** 回滚到之前的视觉方案。

### 2.3 验收标准

#### 场景卡片基础字段

- [ ] 自动从大纲 `locations[]` 创建场景卡片。
- [ ] 字段包括：场景名称、描述、出现频率（高/中/低）、空间类型（室内/室外/半室外）、整体风格、主色调、光线类型、关键道具。
- [ ] 所有字段支持编辑，编辑后触发版本快照。
- [ ] 场景状态：`draft` → `confirmed`。

#### 变体配置

- [ ] 支持时间段变体：日间、夜晚、黄昏。
- [ ] 支持天气变体：晴天、雨天、阴天。
- [ ] 变体可启用/禁用，禁用后分镜节点无法选择。
- [ ] 每个变体独立存储 `image`、`prompt`、`seed`。

#### 基准图生成

- [ ] 系统组装 Prompt：场景描述 + 光线 + 风格 + `no people, no characters, empty room` + `architectural photography`。
- [ ] 固定负向 Prompt：`people, humans, characters, blurry, bad quality, distorted`。
- [ ] 同时生成 3 张候选图（不同 Seed），用户选择最满意的一张。
- [ ] 确认后记录完整 Prompt + Seed，以后同场景生成强制使用此 Seed。

#### 变体图生成

- [ ] 修改光线参数，保持其他不变重新生成。
- [ ] 确认后存入 `variants` 字典。
- [ ] 变体图生成时基于基准 Seed 进行小偏移，确保空间一致。

#### Prompt 管理器

- [ ] 可视化编辑器拆解为 5 层：环境层、光线层、风格层、细节层、负向层。
- [ ] 每层独立编辑，系统自动拼接为完整 Prompt。
- [ ] 修改后可预览拼接结果再决定是否生成。

#### 场景选用规则

- [ ] 分镜节点指定 `scene_id` + `time_of_day`，自动选取对应变体图。
- [ ] 若变体未生成，提示用户先去补充。
- [ ] 同场景所有节点图像生成强制使用 `base_seed`（或变体 seed），不允许随机 seed。

#### 版本快照

- [ ] 所有字段支持版本快照，可回滚到任意历史版本。
- [ ] 每次 AI 生成、用户编辑、变体生成均创建新版本。

### 2.4 非目标

- 本阶段不实现场景的 3D 建模或扫描。
- 本阶段不实现场景视频（动态背景）生成。
- 本阶段不实现场景与角色的自动合成。

## 3. AI 系统要求

### 3.1 工具要求

- 图像 AI Adapter（Flux、NanoBanana、SDXL 等）。
- Seed 控制和复用能力。
- 负向 Prompt 支持。

### 3.2 评估策略

- 人工评估基准图与变体图的空间一致性（≥ 4/5）。
- 自动化检查：同场景所有节点生成时 Seed 使用正确率 100%。
- Prompt 分层编辑后，生成结果与预览一致率 ≥ 90%。

## 4. 技术规格

### 4.1 架构概述

```
大纲确认 → 自动创建场景卡片 → 生成基准图（选 Seed）→ 生成变体图 → 分镜节点按 scene_id + variant 引用
```

### 4.2 集成点

- **API**: `GET/PUT /api/v1/projects/{id}/locations`、`POST /api/v1/projects/{id}/locations/{loc_id}/generate-base`、`POST /api/v1/projects/{id}/locations/{loc_id}/generate-variant`。
- **数据库**: `locations` 表或项目 JSONB 中的 `asset_library.locations`。
- **对象存储**: MinIO/S3 存储场景图。
- **AI 适配器层**: `ImageAdapterPool`。

### 4.3 安全与隐私

- 场景描述和 Prompt 进行敏感词过滤。
- 场景图片按项目隔离存储。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 自动创建场景卡片、基准图生成、变体图生成、场景选用规则。
- **v1.1**: Prompt 分层管理器、版本快照。
- **v2.0**: 智能变体推荐、场景光线自动匹配。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 同场景不同 Seed 导致风格漂移 | 高 | 强制使用 base_seed，变体使用 seed 偏移 |
| 变体图生成遗漏人物 | 中 | 强负向 Prompt + 生成后自动检测 |
| 大量场景变体存储成本高 | 中 | 变体按需生成，未使用变体不预生成 |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成。
- 被依赖模块：单集脚本生成、分镜节点化、分镜图生成、视频片段生成。

## 7. 验收测试要点

1. 基于测试大纲生成 3 个场景，验证卡片字段完整性。
2. 为一个场景生成基准图，验证 3 张候选图展示和 Seed 锁定。
3. 生成日间/夜晚变体，验证变体图存储和 Seed 偏移。
4. 测试 Prompt 分层编辑，验证拼接预览正确。
5. 在分镜节点选择未生成变体，验证系统提示补充。
6. 修改场景描述，验证版本快照生成和回滚。
