# v1.0 分镜图生成模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

分镜节点需要视觉参考图来指导后续视频生成。手工绘制分镜图耗时且成本高，创作者需要 AI 自动生成符合角色一致性、场景一致性和镜头要求的参考图。

### 1.2 解决方案

为每个节点自动组装 6 层 Prompt，注入角色参考图（IP-Adapter）、场景参考图（Seed 锁定）、镜头和运镜描述，调用图像 AI 生成分镜图。提供批量生成、单节点生成、审核通过/重做/上传等能力。

### 1.3 成功标准

- 单节点分镜图生成成功率 ≥ 85%。
- 角色注入成功率 100%（已确认角色）。
- 场景 Seed 锁定执行率 100%。
- 高风险节点自动优化后达标率 ≥ 70%（评分 ≥ 0.85）。

## 2. 用户体验与功能

### 2.1 用户画像

- **视觉导演**：需要快速看到每个镜头的视觉预览。
- **美术指导**：需要确保角色和场景在所有镜头中一致。

### 2.2 用户故事

1. **作为** 视觉导演，**我希望** 为单个节点生成分镜图，**以便** 快速预览镜头效果。
2. **作为** 视觉导演，**我希望** 批量生成所有节点的分镜图，**以便** 高效完成视觉预览。
3. **作为** 美术指导，**我希望** 对分镜图进行通过/重做/上传操作，**以便** 控制进入视频生成的素材质量。
4. **作为** 视觉导演，**我希望** 调整 visual_desc 后重新生成，**以便** 获得更符合预期的画面。
5. **作为** 美术指导，**我希望** 批量选中节点进行批量审核，**以便** 提高工作效率。

### 2.3 验收标准

#### 6 层 Prompt 自动组装

- [ ] 镜头描述层：`visual_desc` 翻译 + `shot_type` 追加 + `camera_move` 追加。
- [ ] 角色 conditioning：出场角色的参考图集 ID 逐一注入 IP-Adapter，指定服装变体则额外注入。
- [ ] 场景 conditioning：场景基准图/变体图作为参考，追加 `base_prompt`，使用 `base_seed` ± 小偏移。
- [ ] 风格统一层：项目配置统一，如 `cinematic, film still, dramatic lighting, 8k`。
- [ ] 质量控制层：`high quality, detailed, sharp focus, professional photography`。
- [ ] 负向 Prompt：系统固定 + 用户自定义。

#### 批量生成与审核

- [ ] 「全部生成」：后台并行队列，逐节点完成即展示，不需等全部完成。
- [ ] 「生成此节点」：单节点生成。
- [ ] 审核操作：✓ 通过（解锁视频生成）、↻ 重新生成（相同参数，随机 Seed 重试）、✏️ 修改后重生（调整 visual_desc 或追加 prompt）、📎 手动上传（跳过 AI 生成）。
- [ ] 所有节点图像通过后解锁「批量生成视频」。

#### 网格/时间轴展示

- [ ] 分镜图以网格或时间轴展示。
- [ ] 每张图上悬浮操作按钮（通过/重做/上传）。
- [ ] 批量操作支持 Shift 多选。

#### 智能 Prompt 精炼（可选增值）

- [ ] 高风险节点（特写/多角色同框）开启自动迭代优化。
- [ ] 循环：生成图像 → 一致性评分（角色/场景/镜头匹配度）→ 若 < 0.85 则 LLM 优化 Prompt → 再生成 → 最多 3 次。
- [ ] 达标或达次数上限后返回最优结果。

#### 一致性保障

- [ ] 所有涉及角色的图像生成必须注入角色参考图集。
- [ ] 分镜图审核不通过不得进入视频生成。
- [ ] 多角色同框时各角色 IP-Adapter 权重独立注入，超过 3 人提示风险。

### 2.4 非目标

- 本阶段不实现视频片段生成（属于下一模块）。
- 本阶段不实现分镜图自动动画化。
- 本阶段不实现分镜图的手绘风格转换（除非通过 Prompt 自定义）。

## 3. AI 系统要求

### 3.1 工具要求

- 图像 AI Adapter（Flux、NanoBanana、SDXL 等）。
- IP-Adapter / ControlNet 参考图注入。
- Seed 控制能力。
- 文本 AI Adapter（用于 Prompt 精炼，可选）。

### 3.2 评估策略

- 人工评估 30 张分镜图：角色一致性、场景一致性、镜头匹配度。
- 自动化一致性评分：角色 ≥ 0.85、场景 ≥ 0.85、镜头 ≥ 0.80。
- 生成失败率统计：目标 ≤ 15%。

## 4. 技术规格

### 4.1 架构概述

```
节点就绪 → StoryboardService 组装 Prompt（角色+场景+镜头+风格+质量+负向）→ ImageAdapter.generateImage() → 返回分镜图 → 审核 → 通过/重做
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/episodes/{ep_id}/storyboard/nodes/generate`、`POST /api/v1/projects/{id}/episodes/{ep_id}/storyboard/nodes/{node_id}/generate`、`PUT /api/v1/projects/{id}/episodes/{ep_id}/storyboard/nodes/{node_id}/review`。
- **数据库**: `storyboard_nodes` 表的 `storyboard_image`、`status`、`ai_model_used` 字段。
- **对象存储**: MinIO/S3 存储分镜图。
- **AI 适配器层**: `ImageAdapterPool`。

### 4.3 安全与隐私

- 图像生成 Prompt 进行敏感词过滤。
- 生成图片按项目隔离存储。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 单节点/批量生成、基础审核、角色/场景一致性注入。
- **v1.1**: Prompt 精炼循环、高风险节点自动优化、批量审核。
- **v2.0**: 分镜图风格迁移、智能构图建议。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 多角色同框一致性差 | 高 | 降低权重、提示风险、允许手动上传参考 |
| 场景 Seed 未生效导致漂移 | 高 | 强制校验 seed 参数传递 |
| 图像生成队列拥堵 | 中 | 并发控制 + 后台队列 + 断点续传 |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成、角色圣经、场景圣经、单集脚本生成、分镜节点化。
- 被依赖模块：TTS 配音生成、视频片段生成。

## 7. 验收测试要点

1. 为 5 个节点生成分镜图，验证字段完整性和图片 URL 存储。
2. 验证角色 IP-Adapter 注入参数正确（查看日志或元数据）。
3. 验证场景 Seed 锁定（同场景不同节点使用相同 base_seed）。
4. 测试审核操作：通过、重做、手动上传。
5. 构造特写/多角色高风险节点，验证 Prompt 精炼循环。
6. 测试批量审核：Shift 多选后批量通过。
