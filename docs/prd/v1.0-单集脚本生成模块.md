# v1.0 单集脚本生成模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

将大纲转化为可拍摄的单集脚本需要大量编剧工作，包括场景划分、角色调度、台词撰写和结尾状态设计。创作者需要 AI 辅助生成初稿，同时保持与角色圣经、场景圣经和叙事连续性的对齐。

### 1.2 解决方案

用户在集详情页点击「生成本集脚本」后，系统自动组装上下文（本集摘要、上集 end_state、角色圣经、场景圣经、时长目标），调用文本 AI 输出结构化脚本。提供场景级编辑、台词编辑、重新生成场景和版本快照能力。

### 1.3 成功标准

- 单集脚本生成成功率 ≥ 90%。
- 输出 JSON 结构合规率 ≥ 95%。
- 上集 end_state 传递成功率 100%（除第一集外）。
- 台词字数与时长估算误差 ≤ 20%。

## 2. 用户体验与功能

### 2.1 用户画像

- **编剧**：需要快速获得场景结构和台词初稿，然后精修。
- **导演**：关注角色调度、场景切换和情绪节奏。

### 2.2 用户故事

1. **作为** 编剧，**我希望** 为单集生成脚本，**以便** 获得场景结构和台词初稿。
2. **作为** 编剧，**我希望** 以场景为单位折叠查看脚本，**以便** 聚焦当前场景。
3. **作为** 编剧，**我希望** 直接编辑任意台词（角色、文本、情绪），**以便** 调整对话细节。
4. **作为** 编剧，**我希望** 重新生成单个场景，**以便** 优化局部而不破坏整集。
5. **作为** 导演，**我希望** 查看本集结尾状态快照，**以便** 确保与下一集衔接。

### 2.3 验收标准

#### 生成触发

- [ ] 集详情页提供「生成本集脚本」按钮，仅在脚本未生成或状态允许时可用。
- [ ] 自动组装上下文：本集摘要、上集 `end_state`、角色圣经信息、场景圣经信息、`featured_characters`/`locations` 约束、时长目标。

#### 输出结构

- [ ] AI 输出必须包含：`episode_title`、`scenes[]`、`end_state`。
- [ ] 每个 `scene` 必须包含：`scene_id`、`location_id`、`time_of_day`、`weather`、`characters_present`、`scene_summary`、`beats[]`、`dialogues[]`。
- [ ] 每个 `dialogue` 必须包含：`char_id`、`text`、`emotion`、`note`（可选）。
- [ ] `end_state` 必须包含：角色情绪/位置、未解决冲突、关键道具状态。

#### 用户交互

- [ ] 以场景为单位折叠展示脚本。
- [ ] 展开后显示完整台词列表：角色名、台词文本、情绪标注。
- [ ] 点击任意台词行直接编辑（角色、文本、情绪）。
- [ ] 可修改场景摘要、时间段、天气。
- [ ] 对整个场景不满意可「重新生成此场景」。
- [ ] 底部显示估算时长（根据台词字数和场景数量粗算）。
- [ ] 确认后进入分镜节点化。

#### 叙事连续性

- [ ] 每集脚本生成强制传入上一集 `end_state`，保持人物状态/情绪/位置连续。
- [ ] 维护关键道具状态字典，脚本 AI 生成时须保持连续性。
- [ ] 第一集 `end_state` 为 null 或初始状态。

#### 版本快照与变更传播

- [ ] 所有编辑支持版本快照。
- [ ] 用户修改后实时弹窗提示影响范围（轻量/中度/深度编辑）。
- [ ] 脚本修改后，已生成的分镜节点标记为「需检查」。

### 2.4 非目标

- 本阶段不实现自动将脚本拆分为分镜节点（需用户确认后触发）。
- 本阶段不实现台词自动对口型。
- 本阶段不实现多语言翻译。

## 3. AI 系统要求

### 3.1 工具要求

- 文本 AI Adapter（Kimi、DeepSeek、GPT4o 等）。
- JSON Schema 约束输出。
- 上下文组装模板。

### 3.2 评估策略

- 人工评估 10 集脚本：场景结构合理性、台词自然度、end_state 完整性。
- 自动化检查：JSON 字段覆盖率、角色/场景引用合法性。
- 叙事连续性检查：相邻集 end_state 传递一致性 100%。

## 4. 技术规格

### 4.1 架构概述

```
用户点击生成 → EpisodeService 组装上下文（大纲+角色+场景+上集end_state）→ TextAdapter.generateText() → 解析存储 → 场景化编辑器
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/episodes/{ep_id}/script/generate`、`PUT /api/v1/projects/{id}/episodes/{ep_id}/script`、`POST /api/v1/projects/{id}/episodes/{ep_id}/script/regenerate-scene`。
- **数据库**: `episodes` 表的 `script` 字段（JSONB），`version_snapshots` 表。
- **AI 适配器层**: `TextAdapterPool`。

### 4.3 安全与隐私

- 脚本内容按项目隔离。
- AI Prompt 不暴露其他用户数据。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 单集脚本生成、场景展示、台词编辑、确认进入分镜。
- **v1.1**: 单场景重生成、end_state 连续性检查、时长估算。
- **v2.0**: 台词情绪自动标注、AI 辅助润色、多版本剧本对比。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 上下文过长超出模型限制 | 高 | 仅传入 featured 角色/场景，非全量圣经 |
| AI 输出 JSON 不合法 | 高 | Schema + 重试 + 后处理修复 |
| end_state 传递错误导致剧情断层 | 高 | 强校验 + 可视化 diff |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成、角色圣经、场景圣经。
- 被依赖模块：分镜节点化、分镜图生成、TTS 配音生成、配乐生成、视频片段生成、合成输出。

## 7. 验收测试要点

1. 为 3 集生成脚本，验证 JSON 结构和字段完整性。
2. 测试第二集生成时是否正确传入第一集 end_state。
3. 修改一句台词，验证版本快照和分镜节点标记为「需检查」。
4. 重新生成单个场景，验证仅目标场景变化。
5. 验证时长估算公式（台词字数 + 场景数）误差 ≤ 20%。
