# v1.0 一致性保障机制 PRD

## 1. 执行摘要

### 1.1 问题陈述

AI 生成的短剧需要保证角色、场景和叙事在多个镜头和集数之间保持一致。没有系统化的一致性保障，容易出现同一角色外貌变化、同一场景风格漂移、剧情前后矛盾等问题。

### 1.2 解决方案

建立角色一致性、场景一致性、叙事连续性三线保障机制。通过 IP-Adapter 参考图注入、Seed 锁定、end_state 传递、剧本医生自洽性检查等手段，在所有生成节点中强制执行一致性规则，并在关键卡口设置审核。

### 1.3 成功标准

- 角色 IP-Adapter 注入执行率 100%（已确认角色）。
- 同场景 Seed 锁定执行率 100%。
- 相邻集 end_state 传递成功率 100%。
- 剧本医生红色错误修复前不允许进入下一阶段，拦截率 100%。

## 2. 用户体验与功能

### 2.1 用户画像

- **视觉导演**：确保角色和场景在所有镜头中一致。
- **编剧/监制**：确保剧情逻辑和叙事连续。

### 2.2 用户故事

1. **作为** 视觉导演，**我希望** 所有涉及角色的图像生成都注入参考图，**以便** 保持角色形象一致。
2. **作为** 视觉导演，**我希望** 同一场景使用相同的 Seed，**以便** 避免风格漂移。
3. **作为** 编剧，**我希望** 每集脚本都继承上集结尾状态，**以便** 保持叙事连续。
4. **作为** 监制，**我希望** 大纲确认前看到自洽性检查报告，**以便** 避免剧情逻辑错误。

### 2.3 验收标准

#### 角色一致性

- [ ] **IP-Adapter 注入**：所有涉及角色的图像生成必须注入角色参考图集，不允许依赖文字描述。
- [ ] **视频人脸锁定**：视频生成时开启 Face Enhancement，首帧人脸特征锁定为生成参考。
- [ ] **服装显式指定**：分镜节点中 `costume_variant` 字段必须填写，禁止 AI 自由选择。
- [ ] **分镜图审核卡口**：不通过不得进入视频生成。
- [ ] **多角色同框**：各角色 IP-Adapter 权重独立注入，超过 3 人提示一致性风险，建议使用远景或群像镜头。

#### 场景一致性

- [ ] **Seed 锁定**：存储 `base_prompt` + `base_seed`，同场景所有生成强制使用相同 Seed。
- [ ] **变体预生成**：不同时间段预生成独立变体图，节点显式指定使用哪个变体。
- [ ] **场景参考图**：基准图审核后作为 conditioning，抑制随机道具变化。
- [ ] **匹配帧过渡**：跨场景切换提供视觉缓冲。

#### 叙事连续性

- [ ] **end_state 传递**：每集脚本生成强制传入上一集 `end_state`，保持人物状态/情绪/位置连续。
- [ ] **key_props 追踪**：维护关键道具状态字典，脚本 AI 生成时须保持连续性。
- [ ] **剧情时间线可视化**：大纲编辑页提供时间线视图，供用户检查逻辑。

#### 剧本医生自洽性检查

- [ ] 大纲确认前强制执行。
- [ ] 检查项 1：角色位置矛盾（同一角色同一时间出现在两个地点）。
- [ ] 检查项 2：道具状态追踪一致性。
- [ ] 检查项 3：情绪曲线合理性（不能连续 5 集无起伏）。
- [ ] 检查项 4：服装连续性。
- [ ] 输出检查报告：红色=错误必须修复，黄色=警告建议修复，绿色=通过。
- [ ] 存在红色错误时不允许进入下一阶段。

#### 变更影响提示

- [ ] 用户修改字段后，顶部横幅提示受影响范围（如"3 个节点需重新生成分镜图"）。
- [ ] 提供「一键标记」和「忽略」按钮。
- [ ] 根据编辑类型自动分类影响级别：轻量/中度/深度。

### 2.4 非目标

- 本阶段不实现自动修复不一致（仅提示和拦截）。
- 本阶段不实现跨项目的一致性复用。
- 本阶段不实现 AI 自动剧情一致性推理（超出规则检查范围）。

## 3. AI 系统要求

### 3.1 工具要求

- 文本 AI Adapter（用于剧本医生检查和剧情推理）。
- 图像/视频 AI Adapter 的参考图注入能力。
- 一致性评分模型（可选，用于分镜图质量检测）。

### 3.2 评估策略

- 构造 10 组不一致数据，验证剧本医生检查拦截率 100%。
- 人工评估 20 集短剧的角色/场景一致性，评分 ≥ 4/5。
- 自动化检查：IP-Adapter 注入率、Seed 锁定率、end_state 传递率均为 100%。

## 4. 技术规格

### 4.1 架构概述

```
角色圣经 → IP-Adapter 参考图
场景圣经 → base_seed + 变体图
上集 end_state → 下集脚本生成
剧本医生 → 大纲/脚本确认前拦截
变更影响 → 用户编辑后实时提示
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/consistency/check-outline`、`POST /api/v1/projects/{id}/consistency/impact-analysis`。
- **数据库**: 复用角色、场景、脚本、节点数据。
- **AI 适配器层**: `TextAdapterPool`、`ImageAdapterPool`、`VideoAdapterPool`。

### 4.3 安全与隐私

- 一致性检查不泄露用户创意内容。
- 参考图和 Seed 按项目隔离。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: IP-Adapter 注入、Seed 锁定、end_state 传递、剧本医生基础检查。
- **v1.1**: 多角色同框风险提示、变更影响分析、分镜图一致性评分。
- **v2.0**: 自动一致性修复建议、跨集叙事逻辑推理、全局一致性仪表盘。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| IP-Adapter 权重不当导致角色崩坏 | 高 | 提供默认权重，允许用户微调 |
| Seed 锁定后变体图 still 不一致 | 中 | 变体使用 base_seed 小偏移 + 强负向 Prompt |
| 剧本医生误报影响用户体验 | 中 | 允许用户标记误报并忽略 |
| end_state 数据结构变化导致传递失败 | 高 | 强 Schema 校验 + 版本兼容 |

## 6. 依赖关系

- 前置依赖：AI 适配器层、版本快照系统、项目创建、大纲生成、角色圣经、场景圣经、单集脚本生成。
- 被依赖模块：分镜图生成、视频片段生成、合成输出。

## 7. 验收测试要点

1. 构造同一角色未注入参考图的生成请求，验证系统强制注入。
2. 构造同场景随机 Seed 请求，验证系统强制使用 base_seed。
3. 生成第二集脚本，验证正确传入第一集 end_state。
4. 构造角色位置矛盾大纲，验证剧本医生红色错误并拦截。
5. 修改角色服装描述，验证顶部横幅提示受影响节点。
6. 多角色同框（4 人）时，验证系统提示一致性风险。
