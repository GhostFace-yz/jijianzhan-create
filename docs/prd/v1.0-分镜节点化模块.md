# v1.0 分镜节点化模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

脚本需要被拆分为可执行的 5-8 秒视频片段单元（节点），每个节点包含镜头类型、运镜、角色、服装、场景变体、情绪、音乐等信息。手动拆分工作量大，且难以保证总时长符合目标。

### 1.2 解决方案

用户确认脚本后，系统自动将脚本拆分为分镜节点，建议镜头类型、运镜方式、情绪标签和时长。提供泳道时间轴编辑器，支持拖拽排序、字段编辑、插入/删除/分裂节点，并实时提示字段修改的影响范围。

### 1.3 成功标准

- 脚本自动拆分为节点的成功率 ≥ 90%。
- 单集节点总时长与目标时长误差 ≤ 15%。
- 节点编辑器拖拽排序和批量操作响应时间 ≤ 200ms。
- 字段修改影响范围提示覆盖率 100%。

## 2. 用户体验与功能

### 2.1 用户画像

- **导演/摄影师**：需要精细控制每个镜头的构图和运镜。
- **剪辑师**：关注节点时长、转场和节奏。

### 2.2 用户故事

1. **作为** 导演，**我希望** AI 自动将脚本拆分为分镜节点，**以便** 快速进入镜头设计阶段。
2. **作为** 导演，**我希望** 在泳道时间轴上查看所有节点，**以便** 按场景和时间顺序组织镜头。
3. **作为** 导演，**我希望** 拖拽节点调整顺序，**以便** 优化叙事节奏。
4. **作为** 导演，**我希望** 修改节点字段时看到影响范围提示，**以便** 了解哪些资产需要重生成。
5. **作为** 摄影师，**我希望** 批量修改多个节点的情绪标签或音乐情绪，**以便** 快速统一风格。

### 2.3 验收标准

#### AI 自动拆分

- [ ] 将脚本 `scenes[]` 拆分为 5-8 秒的视频片段单元（节点）。
- [ ] 为每个节点建议 `shot_type`、`camera_move`、`emotion_tag`、`music_mood`。
- [ ] 估算每个节点 `duration_target`，使单集总时长符合目标。
- [ ] 节点 ID 格式：`ep01-n001`。

#### 节点数据结构

- [ ] 每个节点必须包含：`node_id`、`scene_id`、`scene_variant`、`characters[]`、`shot_type`、`camera_move`、`visual_desc`、`dialogue`、`emotion_tag`、`music_mood`、`duration_target`、`transition_in`、`transition_out`、`status`、`version_history`。
- [ ] `characters` 数组元素包含 `char_id` 和 `costume_variant`。
- [ ] `dialogue` 包含 `char_id`、`text`、`emotion` 或 null。

#### 节点编辑器 UI（泳道时间轴）

- [ ] 横轴为时间顺序，纵轴按场景分泳道。
- [ ] 节点小卡片显示：分镜图缩略图（未生成则占位）、节点 ID、镜头类型标签、时长标注、状态图标。
- [ ] 点击节点展开详细编辑面板，修改所有字段。
- [ ] 拖拽节点调整顺序。
- [ ] 右键菜单：插入新节点（前/后）、删除节点、分裂节点（一个拆为两个）。
- [ ] 批量操作：选中多个节点批量修改 `emotion_tag` 或 `music_mood`。

#### 字段级编辑影响分析

- [ ] 轻量编辑（台词、情绪、时长）：仅影响配音/配乐，分镜图不变。
- [ ] 中度编辑（镜头、运镜、场景变体）：影响分镜图+视频，需重生成。
- [ ] 深度编辑（角色、服装、visual_desc）：影响全链路，需重生成。
- [ ] 用户修改后实时弹窗提示影响范围，确认后继续。

#### 版本快照

- [ ] 每个节点支持版本快照，可回滚到任意历史版本。
- [ ] 节点状态：`pending`、`generating`、`completed`、`needs_redo`。

### 2.4 非目标

- 本阶段不实现节点的 AI 自动优化（如 Prompt 精炼）。
- 本阶段不实现节点间的自动转场生成。
- 本阶段不实现节点预览视频生成。

## 3. AI 系统要求

### 3.1 工具要求

- 文本 AI Adapter（用于拆分和建议镜头）。
- JSON Schema 约束输出。

### 3.2 评估策略

- 人工评估 5 集拆分结果：镜头合理性、时长分配、场景覆盖度。
- 自动化检查：节点总时长与目标时长误差、必填字段覆盖率。

## 4. 技术规格

### 4.1 架构概述

```
脚本确认 → StoryboardService 拆分节点 → 存储 nodes[] → 泳道时间轴编辑器 → 用户编辑/调整 → 状态同步
```

### 4.2 集成点

- **API**: `POST /api/v1/projects/{id}/episodes/{ep_id}/storyboard/split`、`GET/PUT /api/v1/projects/{id}/episodes/{ep_id}/storyboard/nodes`、`POST /api/v1/projects/{id}/episodes/{ep_id}/storyboard/nodes/{node_id}/split`。
- **数据库**: `episodes` 表的 `nodes` 字段（JSONB 或独立表）。
- **AI 适配器层**: `TextAdapterPool`。

### 4.3 安全与隐私

- 节点数据按项目隔离。
- 用户编辑输入进行校验。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 自动拆分、泳道时间轴、基础字段编辑、插入/删除/分裂。
- **v1.1**: 字段影响分析提示、批量操作、拖拽排序。
- **v2.0**: AI 辅助节点优化、智能时长调整、节点预览。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 拆分结果不符合导演预期 | 中 | 允许用户大量手动调整，AI 仅提供初稿 |
| 节点数量多导致前端卡顿 | 中 | 虚拟滚动、分页加载、节点懒加载 |
| 时长估算偏差大 | 中 | 基于台词字数和镜头类型综合估算 |

## 6. 依赖关系

- 前置依赖：项目创建、大纲生成、角色圣经、场景圣经、单集脚本生成。
- 被依赖模块：分镜图生成、TTS 配音生成、配乐生成、视频片段生成、合成输出。

## 7. 验收测试要点

1. 为一集脚本自动拆分节点，验证节点数量、时长总和、字段完整性。
2. 拖拽调整节点顺序，验证保存后顺序正确。
3. 分裂一个节点为两个，验证两个节点时长之和等于原节点。
4. 批量修改 3 个节点的 `music_mood`，验证批量保存成功。
5. 修改节点 `visual_desc`，验证系统提示「影响分镜图+视频」。
6. 验证版本快照回滚功能。
