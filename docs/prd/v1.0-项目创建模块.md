# v1.0 项目创建模块 PRD

## 1. 执行摘要

### 1.1 问题陈述

内容创作者需要一个统一的入口来管理他们的短剧项目。当前没有结构化的项目元数据管理系统，导致创意散落、难以追踪进度，也无法为后续 AI 生成提供一致的上下文。

### 1.2 解决方案

提供一个项目创建与管理模块，让用户在首页查看、筛选、搜索项目卡片，并通过表单创建新项目。创建时收集标题、创意描述、题材、目标集数、时长目标、风格标签和创作备注，项目初始状态为 `draft`，不自动触发任何 AI 生成。

### 1.3 成功标准

- 项目创建成功率 ≥ 99%。
- 首页项目列表加载时间 ≤ 500ms（100 个项目以内）。
- 表单字段校验实时反馈，错误提示覆盖率 100%。
- 项目元数据完整率 ≥ 95%（必填字段全部填写）。

## 2. 用户体验与功能

### 2.1 用户画像

- **初级创作者**：有创意但缺乏编剧经验，需要引导式表单。
- **职业编剧**：熟悉短剧制作，需要快速创建和管理多个项目。

### 2.2 用户故事

1. **作为** 内容创作者，**我希望** 在首页看到所有项目的卡片列表，**以便** 快速找到并继续编辑我的短剧项目。
2. **作为** 内容创作者，**我希望** 按项目状态筛选和按最后修改时间排序，**以便** 优先处理活跃项目。
3. **作为** 内容创作者，**我希望** 通过搜索标题快速定位项目，**以便** 在大量项目中找到目标。
4. **作为** 内容创作者，**我希望** 填写项目基本信息后创建项目，**以便** 进入后续的 AI 辅助创作流程。

### 2.3 验收标准

#### 项目列表页

- [ ] 以卡片形式展示项目：标题、题材标签、集数进度、封面图（占位或上传）、状态标签、最后修改时间。
- [ ] 支持按状态筛选：`draft`、`outlining`、`asset_prep`、`producing`、`completed`。
- [ ] 支持按最后修改时间升序/降序排序。
- [ ] 支持按标题关键字搜索，搜索结果即时展示。
- [ ] 空状态时展示引导创建项目的占位页。

#### 新建项目表单

- [ ] 标题：必填，≤ 50 字，实时显示剩余字数。
- [ ] 创意描述：必填，≤ 1000 字，多行文本框。
- [ ] 题材：单选，选项包括 都市情感/古装/悬疑/喜剧/科幻/其他。
- [ ] 目标集数：数字输入，范围 1-100，留空允许 AI 后续建议。
- [ ] 时长目标：单选，3min / 5min / 10min。
- [ ] 风格标签：多选，预设标签包括 写实/漫画风/赛博朋克/国风/小清新/暗黑系，支持自定义输入。
- [ ] 创作备注：多行文本，可选。
- [ ] 提交前对必填字段和格式进行校验，错误信息明确提示。
- [ ] 创建成功后跳转到项目详情页，状态为 `draft`。

#### 数据模型

- [ ] 项目表包含 `project_id`（UUID）、`meta`（JSONB）、`status`、`created_at`、`updated_at`。
- [ ] 所有表预留 `user_id` 和 `team_id`，默认填充系统单用户 ID。
- [ ] 创建时自动生成 `created_at` 和 `updated_at` 时间戳。

### 2.4 非目标

- 本阶段不实现项目封面图的 AI 生成。
- 本阶段不实现团队协作和权限管理。
- 本阶段不实现项目模板和复制功能。
- 本阶段不实现回收站功能。

## 3. AI 系统要求

本模块不涉及 AI 生成。

## 4. 技术规格

### 4.1 架构概述

```
前端项目列表/表单 → API Gateway → ProjectService → PostgreSQL
```

### 4.2 集成点

- **API**: RESTful API，路径 `/api/v1/projects`。
- **数据库**: PostgreSQL，表名 `projects`。
- **认证**: 预留 `Authorization` Header，当前透传不做校验。

### 4.3 安全与隐私

- 输入字段进行长度和类型校验，防止 XSS 和 SQL 注入。
- 不收集敏感个人信息。
- 预留用户隔离字段。

## 5. 风险与路线图

### 5.1 分阶段发布

- **MVP**: 项目列表、新建项目表单、基础数据模型。
- **v1.1**: 项目封面图上传、项目模板、项目搜索优化。
- **v2.0**: 团队协作、回收站、项目统计仪表盘。

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大量项目列表加载慢 | 中 | 分页加载、虚拟滚动、索引优化 |
| 表单字段扩展频繁 | 低 | meta 字段使用 JSONB 存储 |
| 并发创建冲突 | 低 | UUID 主键 + 唯一索引 |

## 6. 依赖关系

- 无前置依赖。
- 被依赖模块：大纲生成、角色圣经、场景圣经、单集脚本生成。

## 7. 验收测试要点

1. 创建 10 个测试项目，验证列表展示、筛选、排序、搜索功能。
2. 测试边界值：标题 50 字、创意描述 1000 字、目标集数 1 和 100。
3. 测试错误输入：超长标题、非法集数，验证校验提示。
4. 验证数据库字段完整性和时间戳更新。
