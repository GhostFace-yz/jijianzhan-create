## 遗留与待确认项

1. **AI 服务 API Key 配置**
   - 当前 `.env` 中已配置 `KIMI_API_KEY` 和 `AGNES_API_KEY`，但消息发送仍返回「AI 服务认证失败」。
   - 建议 Core 检查 API Key 是否有效，或确认 staging 环境是否预期不联调 AI 供应商。

2. **文件上传 E2E**
   - 预签名 URL 流程需进一步验证，但 OSS 配置为占位值，staging 环境可能无法实际直传。

3. **账单历史**
   - FREE 计划无账单，待付费订阅账号验证账单列表。

## 建议

1. **如 staging 环境不联调 AI 供应商**：当前状态可满足基本功能验证需求，建议上线前在生产环境配置有效的 AI API Key。
2. **如需完整验证 AI 对话**：需 Core 确认并修复 AI 服务认证问题后，Shield 补充验证 SSE 流式输出、Markdown 渲染、代码块复制等功能。
3. **如需完整验证文件上传**：需配置有效的 OSS 密钥后补充验证。
